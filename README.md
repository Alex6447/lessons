# Lessons — платформа для репетиторов

MVP образовательной платформы: личные кабинеты репетиторов и учеников, конструктор
курсов, прогресс прохождения, публичная витрина демо-материалов с заявками и
админ-панель, где **все бизнес-параметры настраиваются без релиза**. Построено по плану
из [docs_lessons/proposal.md](docs_lessons/proposal.md).

Поддерживаются **светлая и тёмная темы**.

## Стек

- **Next.js 15** (App Router) + **React 19** + **TypeScript**
- **Tailwind CSS v4** (тема на CSS-переменных, светлая/тёмная)
- **Prisma ORM** — dev: **SQLite** (без внешних сервисов), prod: **PostgreSQL**
- Аутентификация — лёгкая сессия на JWT-cookie (`jose` + `bcryptjs`), роли `tutor / student / admin`
- Мультитенантность — shared DB, tenant-by-row (изоляция по `tenantId`)

## Запуск за 1 клик (Windows, без окна cmd)

В корне проекта есть **`start.vbs`** — тихий лаунчер.

1. **Дважды кликните `start.vbs`.**
2. При **первом** запуске он сам всё подготовит (скрыто, без консольного окна):
   создаст `.env` из `.env.example`, выполнит `npm install`, засеет БД и соберёт проект.
3. Дождётся готовности сервера и **откроет браузер** на `http://localhost:3000`.

Первый запуск идёт дольше (установка зависимостей + сборка). Последующие — быстрые.

- **Остановить сервер:** дважды кликните **`stop.vbs`** (освобождает порт 3000).
- Лаунчер запускает **production-сборку** (`next start`). Если вы перед этим запускали
  dev-режим, каталог `.next` перезаписывается — лаунчер это замечает и автоматически
  пересобирает.

> Почему `.vbs`, а не `.bat`: VBScript умеет стартовать процесс в полностью скрытом окне
> (`WScript.Shell.Run(cmd, 0, False)`), поэтому консоль не появляется.

## Запуск вручную (любая ОС)

```bash
npm install
npm run db:reset   # создаёт SQLite-БД и заполняет демо-данными
npm run dev        # http://localhost:3000
```

Production-режим локально:

```bash
npm run build
npm run start
```

> `.env` уже содержит dev-настройки (SQLite). Если файла нет — скопируйте `.env.example` в `.env`.

## Тёмная и светлая темы

- Переключатель (🌙 / ☀️) есть в шапке кабинетов, на лендинге, витрине и страницах входа.
- Выбор сохраняется в `localStorage`; при первом визите берётся системная тема
  (`prefers-color-scheme`). Тема применяется до отрисовки — без «мигания».
- Технически: семантические CSS-переменные (`--color-bg/surface/border/ink/...`) в
  `src/app/globals.css`, переопределяемые под `html.dark`.

## Хелп по использованию — по ролям

### Публичный посетитель (без входа)

1. **Лендинг** `/` — оффер, преимущества, превью демо-курсов.
2. **Витрина** `/catalog` — каталог опубликованных демо-курсов. Фильтры по предмету и
   поиск по названию.
3. **Страница курса** `/catalog/<id>` — программа (модули → уроки), профиль репетитора и
   **форма заявки**. Отправленная заявка попадает в раздел «Заявки» соответствующего
   репетитора.

### Репетитор (`/tutor`)

1. **Регистрация:** на `/register` — при создании аккаунта автоматически заводится
   отдельный «кабинет» (tenant); вы видите только свои данные.
2. **Обзор** `/tutor` — статистика (курсы, ученики, заявки) и быстрые действия.
3. **Курсы** `/tutor/courses` → **Конструктор курса** `/tutor/courses/<id>`:
   - редактируйте параметры (название, предмет, цель, уровень, возраст, цена);
   - добавляйте **Модули**, внутри них **Уроки**; в уроке — содержание (Markdown) и
     опциональная **Mermaid-диаграмма**;
   - кнопка **Опубликовать** выводит курс в витрину (демо).
4. **Ученики** `/tutor/students` — добавьте ученика (создаётся аккаунт с временным
   паролем) и запишите на курс.
5. **Заявки** `/tutor/leads` — обращения с витрины.
6. **Оплата** `/tutor/billing` — подписка (абонплата) и история платежей учеников с
   разбивкой комиссия/доля репетитора (каркас Этапа 2).

### Ученик (`/student`)

1. Вход выданными репетитором логином/паролем.
2. **Обзор** `/student` — курсы и прогресс.
3. **Мои курсы** `/student/courses` → курс → **урок**: читаете материал и схему, жмёте
   **«Отметить как пройдено»** — прогресс пересчитывается.

### Админ платформы (`/admin`)

1. **Обзор** `/admin` — сводка по платформе.
2. **Settings** `/admin/settings` — все бизнес-параметры (см. следующий раздел).
   Изменения логируются в **журнал изменений** (кто/когда/что).
3. **Тарифы** `/admin/plans` — создание/редактирование планов абонплаты.
4. **Репетиторы** `/admin/tenants` — все рабочие пространства.

## Админ → Settings: справочник параметров

Значения хранятся в таблице `PlatformSettings` и применяются **без релиза**. Параметры,
влияющие на деньги, фиксируются в момент платежа (изменение ставки не пересчитывает
старые платежи). Группы:

| Группа | Примеры параметров |
|--------|--------------------|
| **Монетизация** | ставка комиссии (дефолт `0.03`), база начисления (course/lesson), валюта |
| **Платежи и выплаты** | провайдер, режим (test/боевой), порог и периодичность выплат |
| **Возвраты** | окно отмены, удержание комиссии при возврате |
| **Фискализация (54-ФЗ)** | кто бьёт чек, ставка НДС |
| **ИИ-помощник** | модель, лимиты генераций |
| **Витрина / контент** | модерация, отображение |
| **Юридические документы** | ссылки на оферту, политику ПДн, агентский договор |
| **Брендинг** | название, цвета, логотип |

## Переменные окружения (.env)

| Переменная | Назначение |
|------------|-----------|
| `DATABASE_URL` | строка подключения к БД. Dev: `file:./dev.db` (SQLite) |
| `AUTH_SECRET` | секрет подписи session-JWT. **Сменить в production** (≥ 32 символа) |
| `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY` | ЮKassa (Этап 2, плейсхолдеры) |
| `ANTHROPIC_API_KEY` | Claude API для ИИ-генерации (Этап 3, плейсхолдер) |

`.env` не коммитится (в `.gitignore`); шаблон — `.env.example`.

## Production-развёртывание

1. Поднимите **PostgreSQL** и задайте `DATABASE_URL`.
2. В `prisma/schema.prisma` смените `provider = "sqlite"` на `"postgresql"`.
3. Примените схему: `npx prisma migrate deploy` (или `npx prisma db push`).
4. Засейте справочные данные при необходимости: `npm run db:seed`.
5. Сгенерируйте надёжный `AUTH_SECRET`.
6. `npm run build && npm run start` (или деплой на хостинг с РФ-локацией данных —
   см. proposal §3 про 152-ФЗ).

> Для рынка РФ платежи — через **ЮKassa** со сплитованием; хранение ПДн — в РФ.

## Troubleshooting

- **`Could not find a production build in '.next'`** — после dev-режима каталог `.next`
  не содержит production-сборки. Выполните `npm run build` перед `npm run start`
  (лаунчер делает это сам).
- **`EPERM ... query_engine-windows.dll.node` при сборке** — запущен dev/preview-сервер,
  он держит движок Prisma. Остановите сервер и повторите `npm run build`.
- **Порт 3000 занят** — `stop.vbs`, либо `npx kill-port 3000`.
- **Сбросить демо-данные** — `npm run db:reset`.
- **После входа сразу выкидывает на `/login`** — проверьте, что `AUTH_SECRET` задан и не
  меняется между запусками (иначе старые cookie становятся недействительными).
